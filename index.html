💬 Claude · Claude Code · OMC 란?
1
Claude — Anthropic의 AI 어시스턴트
Anthropic이 만든 대화형 AI입니다. 글쓰기·분석·번역·요약·코딩·아이디어 발상 등 텍스트 기반 업무를 자연어로 지시하면 수행합니다.

웹(claude.ai)에서 바로 사용하거나, API로 서비스에 연동할 수 있습니다.
2
Claude Code — 파일과 터미널을 직접 다루는 AI
Claude를 터미널 · 데스크탑 앱에서 실행하는 도구입니다. 단순 대화를 넘어 PC의 파일을 직접 읽고·쓰고·수정하며 실제 작업을 자동으로 수행합니다.

• 폴더 안의 파일 읽기 / 문서 생성·수정
• 코드 작성·디버그·리팩터링
• 여러 파일을 한 번에 분석해 보고서 작성
3
OMC (oh-my-claudecode) — 전문 명령어 확장 플러그인
Claude Code에 33개 전문 명령어를 추가하는 플러그인입니다. 자동화·팀 협업·반복 실행·에이전트 운영 등 고급 기능을 /명령어 형태로 바로 호출할 수 있게 해줍니다.

💡 이 가이드에서 소개하는 /autopilot, /ralph, /team 등의 명령어는 모두 OMC 기능입니다.
📋 교육 시작 전 사전 준비 체크리스트

교육 당일 원활한 실습을 위해 아래 항목을 미리 준비해 주세요. 설치에 문제가 생기면 담당자에게 미리 문의하시면 빠르게 해결됩니다.

1
✅ Anthropic 계정 만들기
claude.ai 에서 회원가입 후 로그인하세요.

요금제 선택 (둘 중 하나):
• Claude Pro — 월 $20, 개인 사용에 충분
• Claude Max — 월 $100/$200, 대용량 자동화 작업 시 추천

💡 API 키 방식도 가능합니다. console.anthropic.com 에서 발급 후 claude config set api-key [키값] 으로 설정하세요.
2
✅ 소프트웨어 설치
방법 A — 데스크탑 앱 (초보자 추천)
claude.ai/download 에서 Mac 또는 Windows용 앱 다운로드 후 설치

방법 B — 터미널(CLI)
1. nodejs.org 에서 Node.js v18 이상 설치
2. 터미널에서 npm install -g @anthropic-ai/claude-code 실행
3. claude --version 으로 설치 확인

OMC 플러그인 설치 (교육 필수)
Claude Code 실행 후 채팅창에 /omc-setup 입력 → 자동 설치
3
✅ 실습 환경 준비
연습용 폴더 만들기
바탕화면에 claude-practice 폴더를 하나 만들어두세요.
Claude Code는 이 폴더 안의 파일을 읽고 수정합니다.

샘플 파일 준비 (선택사항)
실습에 활용할 파일이 있다면 폴더에 넣어두세요:
• 엑셀 또는 CSV 파일 (데이터 분석 실습용)
• 텍스트 파일이나 Word 문서 (문서 작업 실습용)
• 기존 HTML/코드 파일 (개발 실습용, 없어도 무관)
4
📌 권장 사전 지식 (필수 아님)
아래 내용은 없어도 교육 참여에 문제없습니다. 알고 오시면 더 빠르게 따라올 수 있습니다.

• 터미널(명령 프롬프트) 기본 사용 — cd 폴더명, ls 정도
• 파일/폴더 구조 개념 — 경로가 무엇인지 이해
• Claude.ai 채팅 사용 경험 — 생성 AI와 대화해본 적 있으면 충분

🙋 설치 중 막히는 부분이 있으면 교육 시작 전 담당자에게 문의하세요. 당일 현장 지원도 가능합니다.
🎯 시작하기: 5분 안에 첫 명령 실행
1
Claude Code 설치 확인
터미널(Mac: Spotlight → Terminal, Win: PowerShell)을 열고 claude --version 을 입력하세요.
설치가 안 되어 있다면 npm install -g @anthropic-ai/claude-code 로 설치합니다.
2
OMC 플러그인 설치
Claude Code를 실행한 뒤 채팅창에 /omc-setup 을 입력하면 oh-my-claudecode가 자동으로 설치됩니다.
이 가이드의 모든 명령어(/autopilot, /plan 등)는 OMC가 있어야 동작합니다.
3
작업할 폴더에서 Claude 실행
터미널에서 작업 폴더로 이동 후 claude 를 입력하세요.
예: cd ~/Desktop/내프로젝트 && claude
Claude가 해당 폴더의 파일들을 인식하고 작업합니다.
4
첫 번째 명령 실행해보기
채팅창에 바로 원하는 것을 말하거나 명령어를 사용하세요.
/deep-interview "랜딩 페이지 만들고 싶어" — 모호한 요청을 질문으로 구체화
/autopilot "버튼 클릭 시 팝업 띄우기" — 바로 구현 시작
5
승인 요청에 응답하기
Claude가 파일을 수정할 때마다 승인을 요청합니다.
y + Enter: 허용  |  n: 거부  |  a: 이번 대화 내 같은 종류 모두 허용

⚖️ 앱 vs 터미널, 뭐가 다를까?

Claude Code는 데스크탑 앱(Mac/Windows)과 터미널(CLI) 두 가지 방식으로 사용할 수 있습니다. 둘 다 같은 AI 엔진을 사용하며, 상황에 맞게 선택하면 됩니다.

🖥️ 데스크탑 앱
장점 설치 후 바로 사용, 클릭 기반 UI로 초보자 친화적
장점 파일 드래그&드롭, 이미지 붙여넣기 등 시각적 작업 편리
장점 Figma·Notion·Slack 등 MCP 서버 연동이 UI에서 바로 가능
단점 고급 자동화(/loop, /schedule) 기능 일부 제한
단점 터미널 명령어 직접 실행이 앱 내부에서만 가능
추천: 처음 시작하는 분, 시각적 작업 위주인 분
⌨️ 터미널 (CLI)
장점 모든 명령어·플러그인·자동화 기능 100% 사용 가능
장점 /loop, /schedule, hooks 등 자동화 파이프라인 구축 가능
장점 여러 세션 동시 실행, 스크립트와 연동 등 확장성 뛰어남
단점 터미널 사용법을 어느 정도 알아야 함
단점 이미지/파일 미리보기 등 시각적 기능이 제한적
추천: 자동화를 적극 활용하고 싶은 분, 개발 경험이 있는 분
💡 둘 다 써도 됩니다! 평소에는 앱으로 편하게, 자동화가 필요할 때는 터미널로 전환하는 것이 가장 효율적입니다. 같은 프로젝트 폴더에서 작업하면 CLAUDE.md 설정이 공유됩니다.

📱 데스크탑 앱의 3가지 모드

💬 채팅
용도 일반 대화·질문·답변, 아이디어 브레인스토밍
특징 파일 수정 없이 대화만 가능, 가장 안전한 모드
활용 예 "이 개념 설명해줘", "마케팅 문구 3개 만들어줘"
추천: 정보 검색, 글 작성, 아이디어 정리
🤝 Cowork (협업)
용도 Claude가 백그라운드에서 작업하는 동안 나도 동시 작업
특징 작업 진행 상황을 실시간 확인, 중간에 방향 수정 가능
활용 예 Claude가 코드 짜는 동안 나는 문서 작업 병행
추천: 긴 작업을 맡기고 다른 일을 병행할 때
💻 코드
용도 파일을 직접 읽고 수정, 터미널 명령어 실행
특징 프로젝트 파일에 접근하여 실제 코드 변경 가능
활용 예 "이 버그 고쳐줘", "새 페이지 만들어줘"
추천: 실제 파일 수정·생성이 필요한 개발 작업

👥 에이전트 팀 구성 & 운영

여러 전문 에이전트를 하나의 팀으로 묶어 동시에 작업시키는 기능입니다. 회사에서 프로젝트 팀을 꾸리는 것과 같은 개념으로, 각 에이전트는 자신의 전문 역할을 맡아 병렬로 일합니다.

1
역할 정하기
작업에 필요한 에이전트 역할을 선택합니다.

주요 역할
• planner — 요구사항 분석 · 작업 계획 수립
• architect — 시스템 구조 설계
• executor — 실제 구현 · 파일 수정
• verifier — 결과 검증 · 테스트 확인
• debugger — 오류 분석 · 버그 수정
• code-reviewer — 코드 품질 리뷰
• scientist — 데이터 수집 · 분석
• writer — 문서 · 보고서 작성
• designer — UI/UX 설계
2
팀 구성 명령 형식
/team 인원수:역할 인원수:역할 "작업 내용"

예시
• /team 1:planner "계획 짜줘" — planner 1명
• /team 3:executor "기능 A·B·C 구현" — executor 3명 동시
• /team 1:planner 2:executor 1:verifier "로그인 기능" — 혼합 팀

💡 숫자를 늘리면 같은 역할의 에이전트가 병렬로 투입됩니다. 큰 작업을 빠르게 처리할 수 있습니다.

CLAUDE.md는 뭐고, 어떻게 작성하나요? +
CLAUDE.md는 프로젝트 폴더에 놓는 설명서입니다. Claude가 새 대화를 시작할 때마다 자동으로 읽어서 프로젝트 규칙을 기억합니다. 작성 예시:

# 프로젝트 개요 이 프로젝트는 React + TypeScript 웹앱입니다. ## 규칙 - 한국어로 주석 작성 - 파일 길이 500줄 이하로 유지 - 테스트 필수 ## 파일 구조 src/components/ — UI 컴포넌트 src/utils/ — 유틸리티 함수
Claude에게 "CLAUDE.md 만들어줘" 라고 하거나, /deepinit 을 실행하면 프로젝트를 분석해 자동 생성해줍니다.
/init과 /deepinit은 뭐가 다른가요? +
/init은 Claude Code 기본 내장 명령어로, CLAUDE.md 파일 템플릿을 빠르게 생성합니다. OMC 없이도 사용 가능합니다.

/deepinit은 OMC 명령어로, 프로젝트 코드 전체를 분석해서 CLAUDE.md + AGENTS.md 등 상세 문서를 자동 생성합니다.

추천: 처음이라면 /init으로 가볍게 시작 → 프로젝트가 복잡해지면 /deepinit으로 심층 분석
.claudeignore 파일은 어떻게 만드나요? +
프로젝트 폴더에 .claudeignore 파일을 만들고, Claude가 읽지 않을 파일/폴더를 한 줄씩 적습니다. .gitignore와 같은 형식입니다:

# 민감 정보 .env .env.local credentials.json # 큰 파일/폴더 node_modules/ dist/ *.log
이 파일에 등록된 항목은 Claude가 읽거나 수정하지 않습니다.
.gitignore 파일은 뭐고 언제 만드나요? +
Git에 올리면 안 되는 파일을 자동으로 제외해주는 설정 파일입니다. 없으면 비밀번호·API 키가 GitHub에 공개되거나, 거대한 라이브러리 폴더가 저장소에 통째로 올라갈 수 있습니다.

언제 만드나요?
프로젝트를 시작하자마자, 첫 커밋 전에 만드는 것이 원칙입니다. Claude에게 "이 프로젝트에 맞는 .gitignore 만들어줘"라고 하면 바로 생성해줍니다.

일반적인 .gitignore 예시: .env ← 비밀번호·API 키 node_modules/ ← 설치된 라이브러리 (용량 크고 재설치 가능) .DS_Store ← Mac 시스템 파일 dist/ ← 빌드 결과물 (소스에서 생성 가능) *.log ← 로그 파일
이 목록에 있는 파일은 git add나 git commit을 해도 Git이 자동으로 무시합니다.
Claude Code 비용 구조가 어떻게 되나요? +
Claude Code는 두 가지 방식으로 사용할 수 있습니다:

1. Claude Pro/Max 구독 — 월 $20(Pro) 또는 $100/$200(Max)로 정액 사용. Max 요금제는 Claude Code 무제한에 가까운 사용량을 제공합니다.

2. API 키 사용 — 사용한 토큰만큼 종량제 과금. 입력 토큰과 출력 토큰 단가가 다르며, 모델에 따라 가격이 달라집니다.

에이전트 (Agent)
특정 역할을 맡아 자율적으로 작업하는 AI. 예: Planner 에이전트는 계획만, Critic 에이전트는 검토만 담당.
토큰 (Token)
AI가 텍스트를 처리하는 단위. 대략 한글 1글자 ≈ 1토큰. 토큰이 많을수록 비용이 늘어남.
컨텍스트 (Context)
AI가 현재 대화에서 기억하는 내용의 범위. 대화가 길어질수록 초반 내용을 잊을 수 있음.
MCP 서버
Claude가 Figma, Notion, Slack 같은 외부 서비스와 대화하게 해주는 연결 도구. /mcp-setup으로 설정.
Git
코드 변경 이력을 관리하는 도구. "git으로 되돌려줘" 하면 이전 상태로 복원 가능.
커밋 (Commit)
현재 코드 상태를 git에 저장하는 것. 되돌리기의 기준점이 됨. "git commit".
터미널 (Terminal)
텍스트 명령어로 컴퓨터를 제어하는 프로그램. Mac은 Terminal 앱, Windows는 PowerShell.
Worktree
같은 프로젝트를 여러 독립 폴더에서 동시에 작업할 수 있게 해주는 git 기능. /project-session-manager에서 사용.
프롬프트 (Prompt)
AI에게 보내는 지시나 질문. 구체적일수록 더 좋은 결과가 나옴.
Claude Code
터미널에서 실행하는 Anthropic의 AI 코딩 도구. 파일 읽기/쓰기와 명령 실행이 가능한 점이 일반 Claude와 다름.
OMC (oh-my-claudecode)
Claude Code에 /autopilot, /plan 등 강력한 명령어를 추가해주는 플러그인. /omc-setup으로 설치.
파이프라인 (Pipeline)
여러 단계를 순서대로 자동 실행하는 워크플로우. 예: 계획 → 구현 → 검증을 자동으로 이어서 실행.
CLAUDE.md
프로젝트 폴더에 두는 설정 파일. Claude가 이 파일을 읽고 프로젝트 규칙과 구조를 파악함.
훅 (Hook)
특정 이벤트 발생 시 자동으로 실행되는 명령. 예: Claude가 작업 완료할 때마다 알림 보내기.
스킬 (Skill)
사용자가 만들거나 OMC가 제공하는 커스텀 명령어(/autopilot 같은 것들). /skill로 관리.
